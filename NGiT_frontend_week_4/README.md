List 3 benefits of using Semantic HTML over generic div tags.

1. **Accessibility** — Screen readers and assistive technologies rely on semantic tags such as `<nav>`, `<header>`, `<section>`, `<article>`, `<main>`, `<aside>`, `<summary>`, and `<footer>` to help users navigate a page meaningfully. Unlike a generic `<div>`, which carries no descriptive meaning, these tags clearly communicate the role and purpose of each part of the page.

2. **SEO (Search Engine Optimization)** — Search engines like Google weight content differently based on its context. Wrapping your main content in `<article>` or `<section>` signals importance, while a `<nav>` tells crawlers those are navigation links — helping your page rank better.

3. **Code Readability & Maintainability** — Semantic markup reads like a document, not a soup of nested divs. When you or a teammate revisits the code, `<header>`, `<main>`, and `<aside>` immediately communicate structure, making it far easier to understand, debug, and extend.

Semantic HTML improves accessibility, SEO, and code readability, making your website more professional and standards-compliant.
`<div>` says *nothing* about intent, it groups content together; semantic tags say *everything* and give meaning.
